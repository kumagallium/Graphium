# 記法と操作

provnote のインライン操作は2つの記号に集約される。

---

## # = 意味を付ける（ラベル）

このブロックは「何」であるかを宣言する。`#` を打つとラベル候補のドロップダウンが出現し、1〜2文字 + Enter で確定（3打鍵）。確定するとテキストから `#xxx` が消え、バッジが表示される。

### コアラベル 5種

| 入力 | 内部名 | PROV |
|---|---|---|
| `#手` | [手順] | prov:Activity |
| `#使` | [使用するもの] | prov:Entity + used |
| `#結` | [結果] | prov:Entity + wasGeneratedBy |
| `#属` | [属性] | prov:Entity + 親への関係 |
| `#パ` | [パターン] | 条件テーブルのトリガー（下記参照） |

「使用するもの」は材料だけでなく装置・ソフトウェアも含む（`prov:used` で紐付く全般）。

### [パターン] ラベルの特別な動作

`#パターン` を通常のテーブルに付けると、そのテーブルは「試料/条件の一覧」として認識され、以下のアクションが出現する。ラベルを外せば普通のテーブルに戻る。

1. **試料ノートの一括作成:** テーブルの各行から新規ノートを自動生成。最初の列の値がノートタイトルになり、他の列がメタデータとして引き継がれる。テンプレート適用も可能。
2. **@ 参照の自動挿入:** 一括作成後、テーブルのセルが自動的に `@S-A ノート` の `@` リンクに変わる。
3. **結果列のトランスクルージョン（将来）:** テーブルに「密度」列を追加すると、各試料ノートの `#結果` から値を自動集約する先として機能する。

```
テーブルに #パターン を付ける前:
│ 試料 │ 温度   │
│ S-A  │ 800℃  │
│ S-B  │ 900℃  │

#パターン を付けた後 → 「試料ノートを一括作成」ボタンが出現

一括作成後:
│ 試料      │ 温度   │
│ @S-A ノート │ 800℃  │  ← 自動的に @ リンクに
│ @S-B ノート │ 900℃  │
```

### エイリアス

`ALIAS_MAP` により、ユーザーは好きな呼び方でラベルを付けられる。`#材料` → `[使用するもの]`、`#パターン` → `[試料]` 等。

### フリーラベル

コアラベルにもエイリアスにも一致しないラベル（`#目的`、`#考察` 等）はフリーラベルとして登録される。PROV には変換されないが、バッジは表示され、知識ネットワーク上で見出しの意味を示す。

### 漸進的構造化

同じラベルでも入力形式によって PROV 出力の精度が自動的に変わる。

| 入力 | PROV 出力 |
|---|---|
| 段落 `Cu粉末 1g` | Entity { rdfs:label: "Cu粉末 1g" } |
| テーブル `名前\|量\|純度` | Entity { rdfs:label: "Cu粉末", provnote:量: "1g" } |
| 画像ブロック | Entity { prov:type: "Image" } |

メモ書きレベルでも動作する。テーブルで書いた分だけ構造化の恩恵を受ける。

### 入力の3つの入口

1. `#` テキスト（主要な入口）
2. スラッシュメニュー `/手順`（ブロック + ラベル同時生成）
3. ドラッグハンドル（後付け）

どれも `LabelStore.setLabel(blockId, label)` を呼ぶだけ。

---

## @ = 別の何かを指す（参照）

別のブロックやノートへのリンクを作る。リンクタイプは `"reference"` の1種類のみ（Zettelkasten の「リンクはリンク」原則）。「支持する」「矛盾する」はリンク周辺のテキストが伝える。

### 「アドレス可能」と「候補に出る」は別の概念

すべてのブロックは blockId を持つのでアドレス可能。しかしオートコンプリートの候補に出るのは構造を持つものだけ。

| 対象 | 候補に出るか | 理由 |
|---|---|---|
| ノートタイトル / H2・H3 見出し | 常時 | スコープの「入口」 |
| ラベル付きブロック | ドリルダウン時 + 検索 | ラベルで名前がある |
| メディア（画像・PDF） | ドリルダウン時 + 検索 | ファイル名・ラベルでマッチ |
| ラベルなしの普通の段落 | 出ない | テキスト断片では選びようがない |

ラベルなしブロックは入口3（リンクコピー）でのみアドレス取得可。

### @ 参照の3つの入口

**入口1: ドリルダウン（場所を覚えている）**

`@Cu焼結/` と打つとそのノートの見出し一覧が出る。選ぶとラベル付きブロック一覧へ。各段階で候補は数件〜十数件。

**入口2: @ 内キーワード検索（場所を忘れた）**

`@SEM 800` のようにキーワードを入力すると、全ノート横断でラベル・ファイル名・テキスト・メディア名を検索し、結果を同じドロップダウン内に表示。ドリルダウンと検索は同じ入力欄で共存（`/` を含めばドリルダウン、含まなければ検索）。

**入口3: ブラウズ → リンクコピー → ペースト（じっくり探す）**

アセットブラウザや別ノートを見に行き、「リンクをコピー」→ 元のノートにペースト → 自動的に `@` メンションに変換。

### @ パスの内部表現

パス表記は人間用の表示名。実体は `blockId` + `noteId` で参照を保持する。見出しをリネームしてもリンクは壊れない。
